# ADR-001: Abordagem de produto — Simulador web acadêmico com backend Python e frontend React

## Status

Accepted

## Date

2026-05-30

## Context

O projeto tem como objetivo apoiar uma apresentação acadêmica de até 15 minutos sobre automação industrial. O escopo exige demonstrar: variáveis de processo, P&ID, lista de instrumentos, folha de dados, arquitetura de automação e Matriz de Causa e Efeito. Além disso, a disciplina requer uma demonstração computacional ao vivo da planta industrial simulada.

A equipe precisou decidir qual abordagem de produto seria mais adequada para atender simultaneamente os requisitos acadêmicos, demonstrar automação industrial de forma didática e ser viável dentro do prazo de uma apresentação.

## Decision

O produto será um simulador web acadêmico composto por:

- Backend em Python com FastAPI, responsável por simular a lógica de controle e o comportamento simplificado da planta de tratamento de água industrial.
- Frontend em TypeScript + React com Vite, responsável por exibir uma interface supervisória didática da planta (sinóptico, variáveis, alarmes, cenários).
- Execução completa via Docker Compose.
- Depuração configurada no VS Code para backend e frontend separadamente.
- Todo o código em português do Brasil.

## Alternatives Considered

### Alternativa 1: Script Python standalone com saída no terminal

- **Description**: Um script Python único que simula a planta e imprime os estados no terminal.
- **Pros**: Simples de implementar; sem dependência de frontend.
- **Cons**: Não oferece visualização gráfica da planta; difícil de demonstrar em apresentação; não representa uma interface supervisória industrial.
- **Why rejected**: Não atende o requisito de demonstração computacional com interface visual. Não representa a camada de supervisão que seria uma IHM/SCADA em uma planta real.

### Alternativa 2: Dashboard pronto (Grafana, Node-RED, ou similar)

- **Description**: Uso de uma ferramenta de dashboard existente como Grafana ou Node-RED para visualizar os dados da simulação Python.
- **Pros**: Interface visual pronta; menos código de frontend a escrever.
- **Cons**: Não demonstra domínio técnico de desenvolvimento; não permite personalização do sinóptico industrial; adiciona dependências externas complexas; o código autoral do projeto ficaria escondido.
- **Why rejected**: Não demonstra os requisitos acadêmicos de Clean Code, SOLID e separação de responsabilidades. Não permite controle total sobre a interface da planta.

## Consequences

### Positive

- Atende todos os 5 requisitos da disciplina de automação industrial.
- Permite demonstração ao vivo com interface visual clara, similar a uma tela supervisória real.
- O frontend React representa conceitualmente a IHM/SCADA de uma planta industrial.
- O backend Python representa conceitualmente o CLP e a lógica de controle.
- A separação backend/frontend reforça o conceito de níveis de automação (campo, controle, supervisão).
- Docker Compose facilita a execução em qualquer máquina durante a apresentação.
- Configuração de debug no VS Code permite explicar o código durante a apresentação.

### Negative

- Requer desenvolvimento de dois componentes separados (backend e frontend).
- Requer configuração de Docker e VS Code além do código da planta.
- Mais tempo de desenvolvimento do que uma solução de terminal.

### Risks

- **Prazo acadêmico**: O escopo completo pode ser excessivo se não for quebrado em tasks pequenas. Mitigação: seguir rigorosamente o fluxo Compozy com tasks sequenciais em `.tasks/`.
- **Complexidade do sinóptico**: Representar a planta visualmente no frontend pode consumir mais tempo do que o esperado. Mitigação: usar SVG ou CSS simples sem bibliotecas gráficas complexas.
- **Compatibilidade da .venv**: A `.venv` existente deve ser compatível com as dependências do backend. Mitigação: verificar a versão Python antes de instalar dependências.

## Implementation Notes

O backend Python não deve ser apresentado como substituto de um CLP real. Durante a apresentação, a explicação correta é que o Python representa uma simulação da lógica de controle que, em uma aplicação industrial real, estaria em um CLP ou edge controller industrial.
